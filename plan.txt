=========================================
PROJEKT: IKE TRACKER (Django SaaS)
CEL: Aplikacja dla mnie i kolegów do śledzenia portfela XTB
TECHNOLOGIA: Python, Django, SQLite (start), Bootstrap 5
=========================================

FAZA 0: KONFIGURACJA I ŚRODOWISKO (Czas: 1-2h)
[ ] Zainstaluj Pythona (wersja 3.10 lub nowsza).
[ ] Utwórz folder projektu i wirtualne środowisko:
    > python -m venv venv
    > venv\Scripts\activate (Windows) lub source venv/bin/activate (Mac/Linux)
[ ] Zainstaluj Django:
    > pip install django pandas openpyxl yfinance
[ ] Utwórz projekt:
    > django-admin startproject ike_tracker .
[ ] Utwórz aplikację (serce logiki):
    > python manage.py startapp core
[ ] Uruchom serwer testowy, żeby zobaczyć, czy działa:
    > python manage.py runserver

FAZA 1: BAZA DANYCH (FUNDAMENT) (Czas: 2-3h)
[ ] W pliku core/models.py zdefiniuj tabele:
    1. Asset (Spółka: symbol, nazwa, waluta)
    2. Portfolio (Portfel: powiązany z Userem)
    3. Transaction (Transakcja: data, typ, ilość, cena, xtb_id) -> xtb_id musi być unique!
    4. PriceHistory (Cache cen: asset, data, cena)
[ ] Zarejestruj modele w pliku core/admin.py (żeby widzieć je w panelu admina).
[ ] Wykonaj migracje (przelej kod do bazy):
    > python manage.py makemigrations
    > python manage.py migrate
[ ] Utwórz superusera (konto admina):
    > python manage.py createsuperuser
[ ] Zaloguj się na /admin i dodaj ręcznie jedną spółkę (np. PKN.PL), żeby sprawdzić czy baza żyje.

FAZA 2: IMPORTER EXCELA (LOGIKA) (Czas: 3-5h)
[ ] W pliku core/forms.py stwórz prosty formularz do uploadu pliku (UploadFileForm).
[ ] W pliku core/views.py napisz widok `upload_view`:
    - Przyjmuje plik z requestu.
    - Używa pandas (kod z Colaba) do odczytu.
    - Pętla po wierszach -> tworzenie obiektów Transaction w bazie.
    - WAŻNE: Obsługa duplikatów (nie dodawaj transakcji, która już ma to samo ID).
[ ] Podepnij widok pod URL w core/urls.py.
[ ] Stwórz prosty template upload.html (tylko przycisk "Wgraj").

FAZA 3: WIDOK DASHBOARDU (Czas: 3-4h)
[ ] Napisz widok `dashboard_view`:
    - Pobiera transakcje zalogowanego użytkownika.
    - Grupuje je (ile mam akcji Orlenu?).
    - Pobiera AKTUALNE ceny (na razie live z Yahoo przy odświeżeniu - powolne, ale proste).
    - Liczy wartość portfela.
[ ] Stwórz template dashboard.html:
    - Użyj Bootstrapa 5 (CDN) dla tabelki.
    - Wyświetl listę posiadanych akcji i sumę w PLN.

FAZA 4: WYKRESY I OPTYMALIZACJA (Czas: 4-6h)
[ ] Dodaj Chart.js do dashboard.html.
[ ] Przekaż dane z Pythona do JavaScriptu (data i wartość portfela).
[ ] Stwórz "Custom Management Command" w Django (skrypt uruchamiany z konsoli):
    - Ma pobierać ceny wszystkich Assetów z bazy danych przez Yahoo Finance.
    - Ma zapisywać je do tabeli PriceHistory.
[ ] Zmień `dashboard_view`, żeby brał ceny z bazy (PriceHistory), a nie z Yahoo.

FAZA 5: DEPLOY (Dla kolegów) (Czas: 2h)
[ ] Załóż konto na GitHub i wrzuć kod.
[ ] Załóż konto na Railway.app lub Render.com (darmowe tiery).
[ ] Podepnij repozytorium i uruchom.
[ ] Wyślij link kolegom.
